# RTX 4090 — Inference Benchmarks

GPU: NVIDIA GeForce RTX 4090 (24 GB VRAM)
Driver: 570.195.03 | CUDA: 12.8

## Qwen3.5-4B (BF16)

- **Engine**: vLLM 0.17.0
- **VRAM usage**: ~8.6 GiB
- **Context length**: 4096
- **GPU memory utilization**: 0.9
- **Date**: 2026-03-10

### Notes

- Peak throughput ~2,950 tok/s at 64 concurrency, still scaling — higher concurrency likely viable.
- Latency degrades gracefully: 1.3s at c=1 → 2.8s at c=64.
- First run at a new batch size triggers CUDA graph compilation (one-time ~20s warmup).
- Model runs in thinking mode by default (Qwen3.5 emits reasoning tokens). Disable with `--chat-template-kwargs '{"enable_thinking": false}'` via `EXTRA_ARGS`.
- ~16 GiB VRAM headroom remaining — could serve a second model or increase context length.

## Qwen3.5-9B (BF16)

- **Engine**: vLLM 0.17.0 (enforce-eager, no CUDA graphs)
- **VRAM usage**: ~17.7 GiB
- **Context length**: 2048 (reduced from 4096 to fit in VRAM)
- **GPU memory utilization**: 0.95
- **KV cache**: 2.76 GiB (~22k tokens, ~24x concurrency at max context)
- **Date**: 2026-03-10

### Throughput vs Concurrency

Prompt: single user message (~16 tokens). Max output: 128 tokens. Temperature: 0.7.

| Concurrency | Tok/s | Req/s | Avg Latency (s) | Failures |
|-------------|-------|-------|------------------|----------|
| 1 | 29.9 | 0.23 | 4.28 | 0 |
| 2 | 49.4 | 0.39 | 5.19 | 0 |
| 4 | 60.7 | 0.47 | 8.44 | 0 |
| 8 | 217 | 1.70 | 4.71 | 0 |
| 16 | 402 | 3.14 | 5.09 | 0 |
| 24 | 630 | 4.92 | 4.88 | 0 |
| 32 | 838 | 6.55 | 4.89 | 0 |

### Notes

- Peak throughput ~838 tok/s at 32 concurrency — ~3.5x slower than the 4B model.
- Requires `--enforce-eager` and reduced context (2048) to fit in 24GB VRAM. Default config (0.9 util, 4096 ctx) OOMs during CUDA graph compilation.
- Single-request latency is ~4.3s (vs 1.3s for 4B) — the 9B model is noticeably slower for interactive use.
- Latency stays relatively flat from c=8 to c=32, suggesting the bottleneck is compute-bound, not memory-bound.
- This model is a tight fit on RTX 4090. For production 9B serving, an A100 (40/80GB) is recommended.
